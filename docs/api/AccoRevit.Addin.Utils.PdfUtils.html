Remarks

The stamp methods write to a temporary output file and then atomically replace the original input PDF via Replace(string, string, string) to avoid leaving a partially written file on failures.

Temporary files are cleaned up in finally blocks; cleanup failures are intentionally ignored.

These methods operate on file paths (not streams) and will throw if required files are missing.

AddDynamicStampToPdf(string, Dictionary<string, string>)

Creates a stamp PDF dynamically by building text form fields from the supplied values, flattens the fields, and overlays the resulting stamp page onto every page of the input PDF.

public static void AddDynamicStampToPdf(string inputPath, Dictionary<string, string> fieldValues)

Parameters

inputPath string

Path to the input PDF that will be modified in place.

fieldValues Dictionary<string, string>

Field-name/value pairs used to create text fields on the stamp. Keys that are null/empty/whitespace are ignored.

Remarks

The stamp page size is based on the first page of inputPath to reduce placement/clipping issues.

Field names are made unique by suffixing an incrementing index (e.g., Key_0, Key_1) to avoid collisions in the generated AcroForm.

The generated fields are flattened into static content before the stamp is applied to the input PDF.

Exceptions

ArgumentNullException

Thrown when inputPath is null/empty/whitespace.

FileNotFoundException

Thrown when inputPath does not exist.

AddStampToPdfWithFields(string, string, Dictionary<string, string>)

Fills AcroForm fields in an existing stamp PDF, flattens them, and overlays the resulting stamp page onto every page of the input PDF.

public static void AddStampToPdfWithFields(string inputPath, string stampPath, Dictionary<string, string> fieldValues)

Parameters

inputPath string

Path to the input PDF that will be modified in place.

stampPath string

Path to a stamp PDF that contains form fields to be filled and flattened.

fieldValues Dictionary<string, string>

Field-name/value pairs to apply to the stamp PDF prior to flattening. Keys that are null/empty/whitespace are ignored.

Remarks

The method performs these steps:

  1. Open stampPath, set provided field values, then flatten fields into a temp stamp file.
  2. Open the input PDF and overlay the temp stamp page onto each page.
  3. Atomically replace the original input file with the stamped output.

The stamp page is copied once into a reusable iText.Kernel.Pdf.Xobject.PdfFormXObject and reused for all pages for performance.

Exceptions

ArgumentNullException

Thrown when inputPath or stampPath is null/empty/whitespace.

FileNotFoundException

Thrown when inputPath or stampPath does not exist.
