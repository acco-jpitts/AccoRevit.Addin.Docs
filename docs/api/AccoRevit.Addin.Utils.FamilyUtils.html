Remarks

These utilities centralize common family-loading patterns for add-ins, including:

  • Reusing already-loaded Autodesk.Revit.DB.FamilySymbol instances when available.
  • Loading a family from an .rfa file using consistent overwrite behavior.
  • Activating symbols before returning them for placement.

Several public methods start their own Autodesk.Revit.DB.Transaction or Autodesk.Revit.DB.TransactionGroup. Do not call them inside an already-active transaction unless you intend to nest transactions appropriately (generally avoid).

public static FamilySymbol GetOrLoadFamilySymbol(this Document doc, string familyName, string familyType, string path, bool showErrors = true, bool useExistingIfFound = true)

Parameters

doc Document

The target document. If null, the current active document is used (when available via Revit context).

Family name to find/load.

Optional type name to match. If null/empty, the method treats this as "load all types" and returns the first symbol found.

Full path to the family .rfa file.

useExistingIfFound bool

Remarks

⚠️ Do not call inside an active Transaction. This method opens a transaction to load families and/or activate symbols as needed.

Parameter names are not used; the method relies on family/type names and the supplied file path.

GetOrLoadFamilySymbols(Document, List<FamilyLoadInfo>, bool, bool)

Loads multiple families (and/or reuses existing symbols) and returns a symbol lookup keyed by Key.

public static Dictionary<string, FamilySymbol> GetOrLoadFamilySymbols(this Document doc, List<FamilyLoadInfo> familyInfos, bool showErrors = true, bool useExistingIfFound = true)

Parameters

doc Document

The target document.

familyInfos List<FamilyLoadInfo>

List of families/types to load. When FamilyType is empty, all types are loaded and additional keys of the form "FamilyName:TypeName" are populated.

showErrors bool

If true, shows a consolidated warning message for any missing/failed loads.

useExistingIfFound bool

If true, reuses an existing matching symbol instead of loading from disk.

Returns

Dictionary<string, FamilySymbol>

A dictionary mapping keys to resolved Autodesk.Revit.DB.FamilySymbol instances. Values may be null when a family/type could not be resolved.

Remarks

⚠️ Do not call inside an active Transaction. This method uses a Autodesk.Revit.DB.TransactionGroup and creates a transaction per family load/activation.

When loading all types for a family, the dictionary contains:

  • info.Key mapped to the first loaded symbol.
  • "FamilyName:TypeName" mapped to each loaded symbol.
