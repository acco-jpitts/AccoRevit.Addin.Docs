Remarks

This runner loads an XML file, parses rules for a specific tag type, resolves/loads the required tag families and types, activates any inactive family symbols, and then repeatedly prompts the user to pick elements to tag until the selection is canceled.

Rule matching and placement behavior (category filtering, tag head location computation, leader configuration, etc.) is delegated to helpers such as XmlTagUtils, XmlRuleCategorySelectionFilter, and element/rule extension methods (e.g., RuleMatchesElement, ComputeTagHeadPoint, TrySetLeader, GetRuleKey).

Remarks

Returns a warning result when the active view is not valid for tagging, when the XML file path is missing, or when no rules exist for the configured tag type.

Returns an error result if the XML cannot be read/parsed.
